Cheatsheet

Flat list of every Core.* / Skin.* function the framework exposes, grouped by domain. Format: Function(args) — what it does. Use Ctrl+F. This is the "I want to do X, which function?" index — signatures come straight from types/core.lua (the source of truth).

Tag legend: (S) server only · (C) client only · (SC) shared (both sides). Args ending in ? are optional.

"How do I…?" quick answers​

I want to…Use
Notify a playerCore.Notify(msg, kind) (C) / Core.Notify(src, msg, kind) (S)
Get a player objectCore.Player.Get(src) (S) / Core.GetPlayerData() (C)
Give / take moneyplayer.Functions.AddMoney(account, amount) / RemoveMoney(...)
Give / take an itemCore.Inventory.AddItem(id, name, count) / RemoveItem(...)
Check an itemCore.Inventory.HasItem(id, names, amount)
Make an item usableCore.Inventory.CreateUseableItem(name, cb)
Open a stash / shopCore.Inventory.Open(invType, data)
Ask the player for inputCore.Input.text(title) / .number / .select / .confirm (C)
Register a keybindCore.RegisterKey(name, desc, key):onPressed(fn) (C)
Show a menuCore.Menu.show(...) (C)
Show a promptCore.Prompt.createGroup(...) then :addPrompt(...) (C)
Server↔client requestCore.Callback.triggerServer(name, ...) / register(name, fn)
Check admin levelCore.Permissions.HasLevel(src, n) / HasCommandLevel(src, cmd) (S)
Check VIPCore.Vip.HasTier(src, n) (S)
Spawn a static NPC / prop / vehicleCore.Ped.Create / Core.Prop.Create / Core.Vehicle.Create (C)
Put a blip on the mapCore.Blip.Create(data) (C)
Make an interaction pointCore.Interact.Create(data) (C)
Apply a skin / clothesSkin.ApplySkin(skin) / Skin.ApplyClothes(clothes) (C)
Register a config-editable fieldCore.Editor.RegisterField(desc) (SC)
Persist a world object (DB)Core.World.Add(type, data) (S)
Define a job / gang / itemCore.Jobs.Create / Core.Gangs.Create / Core.Items.Create (S)
Log with a toggleable categoryCore.Logger.info(cat, fmt, ...) (SC)

Module system (SC)​

  • Core.RegisterModule(def) — register a framework module; hooks (init, onPlayerLoaded, beforeSpawn, onSave, onPlayerUnloaded) dispatched in dependency order.
  • Core.Modules.dispatch(hook, ...) — fire a hook to every registered module in dependency order.

Sessions (S)​

  • Core.Session.get(src) — session for a source (account, identifiers, state) or nil.
  • Core.Session.waitFor(src, timeout?) — block until the session exists (default 5000 ms).
  • Core.Sessions — table of active sessions indexed by source.
  • Core.Players — table of loaded players indexed by source.

Players (S)​

  • Core.Player.Login(src, charId) — build the Player object and dispatch onPlayerLoaded.
  • Core.Player.Logout(src, skipSave?) — log the character out (dispatch unload, save unless skipSave).
  • Core.Player.Save(src) — persist the character (the only write path).
  • Core.Player.Get(src) — Player object for a source or nil.
  • Core.Player.GetAll() — array of all loaded Players.
  • Core.Player.GetByUserId(userId) — Player by account id (stable across sessions).
  • Core.Player.GetByCitizenId(citizenid) — Player by citizenid (RSG interop).

Player.Functions (on the Player object, S)​

  • player.Functions.AddMoney(account, amount, reason?) — credit an account; returns bool.
  • player.Functions.RemoveMoney(account, amount, reason?) — debit an account; returns bool.
  • player.Functions.SetMoney(account, amount) — set an account balance; returns bool.
  • player.Functions.GetMoney(account) — read an account balance.
  • player.Functions.SetJob(name, grade?) — set job; returns bool.
  • player.Functions.SetGang(name, grade?) — set gang; returns bool.
  • player.Functions.SetMetaData(key, value) — write a metadata key.
  • player.Functions.GetMetaData(key) — read a metadata key.
  • player.Functions.AddItem(item, amount?, slot?, info?) — add an item; returns bool.
  • player.Functions.RemoveItem(item, amount?, slot?) — remove an item; returns bool.
  • player.Functions.Save() — persist the player.
  • player.Functions.Logout() — log the character out.
  • player.Functions.UpdatePlayerData() — push PlayerData to the client mirror.

Accounts (S)​

  • Core.Account.getIdentifiers(src) — cfx identifiers { discord?, license?, steam? }.
  • Core.Account.resolve(identifiers) — load or create the account (Discord first, license fallback).
  • Core.Account.setFavoriteCharacter(userId, charId) — set the favorite character; returns bool.

Characters (S)​

  • Core.Character.create(userId, data) — create a character in DB; returns charId or nil.
  • Core.Character.getAllForUser(userId) — array of the account's characters.
  • Core.Character.saveAppearance(charId, skin, clothes) — save skin + outfit (characters table).
  • Core.Character.softDelete(userId, charId) — soft-delete a character; returns bool.

Permissions (S)​

  • Core.Permissions.GetLevel(src) — admin level 0-50 (users.admin_level).
  • Core.Permissions.HasLevel(src, minLevel) — at least this level? returns bool.
  • Core.Permissions.HasCommandLevel(src, command) — standard admin-command guard (maps Config.Perms.CommandLevels).
  • Core.Permissions.SetLevel(userId, level, bySrc?) — set level; returns bool, err?.
  • Core.Permissions.SetVip(userId, tier, bySrc?) — force a VIP tier (grizzly/coyote/nil); returns bool, err?.
  • Core.Permissions.Refresh(src) — re-apply aces/principals for the player.

VIP (S)​

  • Core.Vip.GetTier(src) — resolved VIP tier 'grizzly'|'coyote'|nil (Discord roles, cached).
  • Core.Vip.HasTier(src, minTier) — compare to numeric rank (coyote=1, grizzly=2); returns bool.

Inventory (S unless noted)​

  • Core.Inventory.AddItem(id, name, count?, metadata?, slot?) — add item; returns bool.
  • Core.Inventory.RemoveItem(id, name, count?, metadata?, slot?) — remove item; returns bool.
  • Core.Inventory.HasItem(id, items, amount?) — has item(s)? items is string or string[]; returns bool.
  • Core.Inventory.GetItemCount(id, name, metadata?) — count of an item.
  • Core.Inventory.CanCarryItem(id, name, count, metadata?) — fits weight/slots? returns bool.
  • Core.Inventory.Open(invType, data) — open an inventory (player|stash|shop|crafting|drop).
  • Core.Inventory.CreateUseableItem(name, callback) — register an item-use callback fn(source, item).
  • Core.Inventory.SetMetadata(id, slot, metadata) — overwrite a slot's metadata.
  • Core.Inventory.Search(id, mode, names, metadata?) — mode = 'slots'|'count'; returns table or integer.
  • Core.Inventory.GetItemList() — all item definitions (data/items), indexed by name.

id = a player's source, or a stash/drop identifier.

Keybinds (C)​

  • Core.RegisterKey(name, desc, defaultKey) — register a remappable key; returns a handle. Use this, never RegisterKeyMapping.
  • handle:onPressed(fn) — run fn on press; returns the handle (chainable).
  • handle:onReleased(fn) — run fn on release; returns the handle.
  • Core.Keys.Register(resource, name, description, defaultKey, category?) — low-level register (prefer Core.RegisterKey).
  • Core.Keys.On(resource, name, event, callback) — bind 'pressed'|'released' to a callback.
  • Core.Keys.Get(resource, name) — current key or nil.
  • Core.Keys.Set(resource, name, key) — override the key.

Input (C)​

  • Core.Input.open(options) — open the input dialog with a free config; returns the value or nil.
  • Core.Input.text(title, opts?) / .number / .password / .textarea — single-field dialogs.
  • Core.Input.slider(title, opts?) / .checkbox / .date / .color — typed fields.
  • Core.Input.select(title, options, opts?) — dropdown; returns the chosen value.
  • Core.Input.confirm(title, message?, opts?) — confirmation; returns bool.
  • Core.Input.native(label, placeholder?, maxLen?) — jo.input.native compat (always a string).
  • Core.Input.IsOpen() — is the input open? returns bool.

Settings (C)​

  • Core.Settings.Register(def, owner?) — persistent player setting shown in the pause menu.
  • Core.Settings.Get(id) — read a setting value.
  • Core.Settings.Set(id, value) — write a setting value.

Pause menu (C)​

  • Core.Pause.IsOpen() — is the pause menu open? returns bool.
  • Core.Pause.Open() — open it.
  • Core.Pause.Close() — close it.
  • Core.Pause.Toggle() — toggle it.

Player data mirror (C)​

  • Core.GetPlayerData() — local PlayerData mirror (updated by events).
  • Core.IsPlayerLoaded() — character loaded? returns bool.
  • Core.OnPlayerLoaded(callback) — run now if loaded, else at lo:Client:OnPlayerLoaded.

HUD (C)​

  • Core.Hud.SetStatus(name, pct) — set a status bar to a percentage.
  • Core.Hud.GetHunger() — current hunger.
  • Core.Hud.GetThirst() — current thirst.
  • Core.Hud.AddHunger(v) — add to hunger.
  • Core.Hud.AddThirst(v) — add to thirst.
  • Core.Hud.SetMinimap(state) — show/hide the minimap.
  • Core.Hud.SetState(name, active) — toggle a named HUD state.
  • Core.Hud.SetVisible(visible) — show/hide the whole HUD.

NUI focus (C)​

  • Core.MenuFocus.Acquire(owner) — push an owner onto the NUI focus stack.
  • Core.MenuFocus.Release(owner) — pop an owner off the focus stack.
  • Core.MenuFocus.HasFocus() — does anything hold focus? returns bool.

Notifications (SC)​

  • Core.Notify(message, kind?) — (C) notify the local player (kind = 'error'|'success'|nil).
  • Core.Notify(src, message, kind?) — (S) notify a player by source.
  • Core.ShowError(resource, msg) — (C) convenience error toast.
  • Core.ShowSuccess(resource, msg) — (C) convenience success toast.

Callbacks (SC)​

  • Core.Callback.register(name, fn) — register a callback (server: source is first arg).
  • Core.Callback.triggerServer(name, ...) — blocking call to the server (returns its result).
  • Core.Callback.triggerClient(name, src, ...) — call a client callback from the server.
  • Core.Callback.register.latent(...) — latent (chunked) callback for large payloads.

Menus (C)​

  • Core.Menu — transparent proxy of jo.menu. Common members: show, send, refresh, isOpen, missingMenuHandler. Use MenuClass:refresh() to update an open menu without losing position.

Prompts (C)​

  • Core.Prompt.createGroup(title, position?) — create a prompt group; returns the group.
  • group:addPrompt(key, label, holdTime, page?) — add a prompt (label accepts HTML); returns the prompt instance.
  • group:display(page?) — show the group this frame.
  • group:hide() — hide the group.
  • prompt:setLabel(label) / :setVisible(bool) / :setEnabled(bool) — mutate a prompt.
  • Core.Prompt.createNative(coords, label, distance, keyName, callback) — native zone prompt (key resolved via Core.PromptKeys, default 'G'); returns an instance with :Destroy().
  • Core.Prompt.createTarget(entity, label, name?, distance?, callback, keyName?) — native prompt targeting an entity (right-click/target by default, or a key if keyName given); returns an instance with :Destroy().
  • Core.Prompt.isCompleted(group, key, fireMultipleTimes?) — true for one frame on press/hold-complete.
  • Core.PromptKeys — map of key name → game control hash (for native prompts).

World primitives (C)​

Blips​

  • Core.Blip.Create(data) — create a map blip (town grouping + proximity); returns handle or nil.
  • Core.Blip.Remove(idOrBlip) — remove a blip.
  • Core.Blip.Get(id) — blip handle by id or nil.
  • Core.Blip.SetName(idOrBlip, name) — rename a blip.

Peds (static NPCs, streamed by proximity)​

  • Core.Ped.Create(data) — spawn a static ped ({ id, model, coords, heading?, scenario?, followPlayer?, variation?, saddle?, targetable?, onSpawned? }); returns id.
  • Core.Ped.Remove(id) — remove a ped.
  • Core.Ped.Get(id) — ped handle or nil.

Props / Vehicles / Horses / Markers / Text3D​

  • Core.Prop.Create(data) / .Remove(id) / .Get(id) — static objects streamed by proximity.
  • Core.Vehicle.Create(data) / .Remove(id) / .Get(id) — static locked/frozen/invincible vehicles.
  • Core.Horse.Create(data) / .Remove(id) / .Get(id) — static horses (Core.Ped + coat + saddle?).
  • Core.Marker.Create(data) / .Remove(id) / .Get(id) — ground markers ({ id, hash, coords, scale?, color?, offset? }).
  • Core.Text3D.Create(data) / .Remove(id) / .Get(id) — floating 3D text ({ id, text, coords, anchor?, zOffset? }).

Interaction point (composition)​

  • Core.Interact.Create(data) — blip + ped/prop + marker + text3d + prompt in one ({ id, coords, blip?, ped?, prop?, marker?, text?, prompt?, onSelect? }). Prompts only.
  • Core.Interact.Remove(id) / Core.Interact.Get(id).

Camera (C)​

  • Core.Camera.Create(opts) — create + activate the scripted camera ({ coords, fov?, point?, orbit?, interp? }); returns handle or nil.
  • Core.Camera.SetOrbit(focus, dist, yaw, pitch) — (re)define orbit; focus is fun(): vector3|nil.
  • Core.Camera.GetOrbit() — { dist, yaw, pitch } or nil.
  • Core.Camera.GetHandle() — native camera handle or nil.
  • Core.Camera.GetCoord() — current position or nil.
  • Core.Camera.SetFov(value) — set FOV (clamped 20..120).
  • Core.Camera.GetFov() — current FOV or nil.
  • Core.Camera.InterpToPoint(point, dist, zoomFov, duration?) — interpolated zoom toward a point.
  • Core.Camera.InterpToOrbit(duration?) — interpolate back to the orbital view.
  • Core.Camera.IsInterpolating() — mid-interpolation? returns bool.
  • Core.Camera.CursorRay(cursorX, cursorY) — ray-pick: returns cam position + direction.
  • Core.Camera.Destroy(interp?) — destroy the camera and return control to gameplay.

Gizmo (C)​

  • Core.Gizmo.Use(entity, cfg?, allowPlace?) — 3D transform editor; returns final position+rotation or nil.
  • Core.Gizmo.Cancel() — cancel the gizmo.

Game / ped helpers (C)​

  • Core.setPVP(enabled) — enable/disable player-vs-player damage.
  • Core.RestoreFullCores(ped?) — refill health/stamina cores to 100 (default PlayerPedId()).
  • Core.FillUpCores() — upgrade max health/stamina tanks per Config.Game.
  • Core.FreezePlayer(state) — freeze/unfreeze the local player (position + invincibility + blocks non-temporary events).
  • Core.Functions — table of misc client core helpers.

Skin / appearance (C)​

  • Skin.ApplySkin(skin, target?) — apply a full skin (jo.component format + lo extensions: head/body hashes, hide upper/lower, whistle, eyes…).
  • Skin.ApplyClothes(clothes, target?) — apply a clothes table table<string, SkinAsset>.
  • Skin.Apply(category, data, target?) — apply one component (shop item hash or meta-tag asset).
  • Skin.Remove(category, target?) — remove a category's component (shop item and meta tag).
  • Skin.RemoveAllClothes(target?) — strip all clothes.
  • Skin.GetComponentList() — full MP component list sorted by sex/category.
  • Skin.IsClothes(category) — is this category a clothes slot? returns bool.
  • Skin.ApplyHead(target?, headIndex, skinTone) — apply a head.
  • Skin.ApplyBody(target?, bodiesIndex, skinTone) — apply a body.
  • Skin.ApplyEyes(target?, eyesIndex) — apply eyes.
  • Skin.ApplyTeeth(target?, teethIndex) — apply teeth.
  • Skin.ApplyHair(target?, data) — apply hair (SkinAsset).
  • Skin.ApplyBeard(target?, category, data) — apply beard (beards_complete|beards_mustache|beards_chops|beards_chin|beards).
  • Skin.ApplyExpression(target?, name, value) — apply a face expression (value -1.0..1.0).
  • Skin.ApplyOverlay(target?, category, data) — apply a face overlay (eyebrow|scar|ageing|freckles|moles|spots|eyeshadow|blush|lipstick|eyeliner|foundation…).
  • Skin.RemoveOverlay(target?, category) — remove an overlay.
  • Skin.ApplyWhistle(target?, whistle) — configure the ped's whistle ({ pitch?, clarity?, shape? }).
  • Skin.SetScale(target?, scale) — set ped scale (0.60-1.20 in the creator).

Config editor (SC)​

  • Core.Editor.RegisterField(desc) — declare an editable config field ({ path, label?, type?, section?, min?, max?, choices?, level?, hot?, validate? }).
  • Core.Editor.OnChange(pattern, fn) — react to changes under a path prefix; fn(path, value, old).
  • Core.Editor.Get(path) — current value at a config path.
  • Core.Editor.GetDefault(path) — default value captured at boot.
  • Core.Editor.GetField(path) — the field descriptor or nil.
  • Core.Editor.Fields() — full registry + ordered keys.
  • Core.Editor.Validate(desc, value) — validate a value against a descriptor; returns bool, err?.
  • Core.Editor.Set(path, value) — apply locally (internal — go through the server to persist).
  • Core.Editor.Discover() — auto-discover Config leaves (called at boot).

World registry — persisted entities (S unless noted)​

  • Core.World.Add(entType, data, createdBy?) — create + persist a world entity (DB world_entities) and sync; returns id or nil.
  • Core.World.Update(id, data) — update an entity; returns bool.
  • Core.World.Remove(id) — delete an entity; returns bool.
  • Core.World.Get(id) — entity by id or nil.
  • Core.World.GetAll() — all entities.
  • Core.World.ShapeInteraction(ent) — shape an interaction entity into the editor-front format.
  • Core.World.InteractionsFor(entityType, entityName) — interactions tied to a job/gang/public.
  • Core.World.OpenPanel(view?) — (C) open the /editor React panel (optional view).
  • Core.World.ClosePanel() — (C) close the panel.

Actions (SC)​

  • Core.Action.register(actionType, handler) — register an interaction action (client: UI dispatch, server: authoritative logic).
  • Core.Action.run(...) — run an action.
  • Core.Action.has(actionType) — is an action registered? returns bool.

Jobs / Gangs (S)​

  • Core.Jobs.Create(name, def) — define a job (extends Core.Shared.Jobs, persisted + synced); returns bool.
  • Core.Jobs.Delete(name) — delete a job; returns bool.
  • Core.Jobs.GetAll() — all jobs.
  • Core.Jobs.Get(name) — one job or nil.
  • Core.Gangs.Create(name, def) / .Delete(name) / .GetAll() / .Get(name) — same for gangs.

Custom items (S)​

  • Core.Items.Create(name, data) — define a custom item (data/custom_items.json, extends the inventory); returns bool.
  • Core.Items.Delete(name) — delete a custom item; returns bool.
  • Core.Items.GetCustom() — table of custom items.

Society accounts (S)​

  • Core.Society.GetBalance(name) — balance of a job's society account.
  • Core.Society.Add(name, amount) — credit a society account; returns bool.
  • Core.Society.Remove(name, amount) — debit a society account; returns bool.

Logger (SC)​

  • Core.Logger.info(category, fmt, ...) — info log (category toggled by Config.Debug / /debug).
  • Core.Logger.warn(category, fmt, ...) — warning log.
  • Core.Logger.error(category, fmt, ...) — error log.
  • Core.Logger.debug(category, fmt, ...) — debug log (only if the category is enabled).

Data tables (read-only)​

  • Core.CharacterData — static module data (creator catalogs: Hairs, Exclusive, Labels, ExtractPeds…).
  • Core.Enums — shared enums (SessionState…).
  • Core.WeaponData.Types — weapon group (GROUP_*) → internal type (SHORTARM/LONGARM/SHOTGUN/MELEE_BLADE/GROUP_BOW).
  • Core.WeaponData.Customizable — weapon types customizable at the workshop.
  • Core.WeaponData.Currencies — currencyType → currency name (cash/gold/bloodmoney).
  • Core.WeaponData.CategoryLabels / .PartLabels / .MaterialLabels — FR labels for component categories / parts / materials.
  • Core.WeaponData.MetalSwatches — metal swatch names by variant index.
