Core — Permissions server

Admin levels 0-50 (users.admin_level). Every admin command is guarded server-side; targeting is by userId, never by source.

Core.Permissions.GetLevel​

Core.Permissions.GetLevel(src)

The player's admin level (0-50).

Parameters​

NameTypeDescription
srcintegerPlayer source.

Returns​

integer

Example​

local level = Core.Permissions.GetLevel(src)

Core.Permissions.HasLevel​

Core.Permissions.HasLevel(src, minLevel)

Does the player have at least this level?

Parameters​

NameTypeDescription
srcintegerPlayer source.
minLevelintegerMinimum level required.

Returns​

boolean

Example​

if Core.Permissions.HasLevel(src, 30) then
    -- staff action
end

Core.Permissions.HasCommandLevel​

Core.Permissions.HasCommandLevel(src, command)

Standard guard for admin commands. Maps Config.Perms.CommandLevels (default MaxLevel).

Parameters​

NameTypeDescription
srcintegerPlayer source.
commandstringCommand name.

Returns​

boolean

Example​

RegisterCommand('giveitem', function(src)
    if not Core.Permissions.HasCommandLevel(src, 'giveitem') then return end
    -- ...
end)

Core.Permissions.SetLevel​

Core.Permissions.SetLevel(userId, level, bySrc?)

Set an account's admin level.

Parameters​

NameTypeDescription
userIdintegerAccount id.
levelintegerNew level.
bySrc?integerSource of the actor (audit).

Returns​

boolean, string | nil — success and an optional error message.

Example​

local ok, err = Core.Permissions.SetLevel(userId, 40, src)

Core.Permissions.SetVip​

Core.Permissions.SetVip(userId, tier, bySrc?)

Force a VIP tier ('grizzly', 'coyote' or nil). Normally resolved by Discord roles — this is the manual override.

Parameters​

NameTypeDescription
userIdintegerAccount id.
tierstring | nil'grizzly' | 'coyote' | nil.
bySrc?integerSource of the actor (audit).

Returns​

boolean, string | nil

Example​

Core.Permissions.SetVip(userId, 'coyote', src)
