Exports

The actual lo exports exposed to external resources (internally, the code goes through Core.*). Call them with exports.lo:Name(...) or exports['lo']:Name(...).

Tags: (S) server · (C) client.

Bridge — RSG interop​

ExportSideDescription
GetCoreObject()C + SRSGCore compat object (Jobs/Gangs/Items/PlayerData on C; Functions/Shared/Config on S).
GetCoreVersion()SVersion string.
ExploitBan(src, reason?)SKicks the player with an optional message.

The bridge also re-exposes all inventory exports in RSG format (CreateInventory, OpenInventory, AddItem, RemoveItem, HasItem, GetItemCount, CreateUseableItem, GetItemList…). See bridge/server.lua.

Players (S) — core/server/player/player.lua​

exports.lo:GetPlayer(src)               -- Player object (Functions.AddMoney/SetJob/SetMetaData/Save…)
exports.lo:GetPlayerByCitizenId(citizenid)
exports.lo:GetPlayerByUserId(userId)
exports.lo:GetPlayers()                 -- Core.Players table

Permissions (S) — core/server/permissions.lua​

exports.lo:getLevel(src)                -- admin level 0-50
exports.lo:hasLevel(src, minLevel)      -- ➝ boolean
exports.lo:setLevel(userId, level, bySrc?)
exports.lo:refreshPerms(src)

VIP (S) — core/server/account/vip.lua​

exports.lo:getVipTier(src)              -- ➝ 'grizzly'|'coyote'|nil
exports.lo:hasVipTier(src, minTier)     -- ➝ boolean
