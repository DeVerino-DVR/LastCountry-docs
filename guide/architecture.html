Architecture

lo is a single resource organized into three spaces:

lo/
├── core/        the kernel — sessions, accounts, players, characters, permissions
├── modules/     the systems — creator, inventory, hud, pause, imaps, voice…
├── web/         the single React app (NUI), split into sections
├── bridge/      RSG compatibility (mirror events + export proxy)
├── data/        hot-mutable state (JSON written by the server)
└── config.lua   single configuration (Config table)

The Session is the core​

Everything is server-authoritative around a per-player state machine:

  1. playerConnecting — identification Discord first, license fallback; loading or creating the account (users).
  2. playerJoining — createSession builds the session, emits lo:session:created, sets the statebags (userId, charId, adminLevel, vip).
  3. The client picks a character → Core.Player.Login(src, charId) builds the Player object (RSG format, interop key citizenid).

Every Player mutation emits a lo:Server:* event that the bridge relays as RSGCore:Server:* — this is how third-party RSG resources work without rsg-core.

Persistence​

A single write path: Core.Player.Save → UPDATE characters. Triggered periodically, on playerDropped, on resource stop, and on txAdmin events. A hot restart rebuilds the sessions from the statebags.

The module registry​

Modules never touch the kernel. They register themselves:

Core.RegisterModule({
    name = 'mymodule',
    dependencies = { 'inventory' },
    init = function() end,
    onPlayerLoaded = function(player) end,
    beforeSpawn = function(player) end,
    onSave = function(player) end,
    onPlayerUnloaded = function(player) end,
})

Hooks are dispatched in topological dependency order from Core.Player.Login/Logout/Save. A module must be removable from the manifest without breaking the framework boot.

Web​

A single React app (web/src), mounted in multi-NUI mode via jo_libs (jo.nui.load). Each section (inventory, hud, pause, ui) listens to its own NUI messages and replies via fetchNui. The NUI protocol is the contract: the Lua evolves without touching the front end as long as the messages stay stable.
