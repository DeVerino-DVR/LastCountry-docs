UI — lo_ui ​

L'interface NUI du framework est une ressource React / TypeScript distincte : resources/[ui]/lo_ui. Elle est pure-web (aucune ui_page ni RegisterNUICallback Lua propre) : c'est lo qui la sert via le frame NUI de jo_libs.

lua
-- resources/[core]/lo/init.lua
jo.nui.load('lo', 'nui://lo_ui/web/build/index.html')

lo et lo_ui sont deux ressources : un changement de front ne nécessite que restart lo_ui.

Build ​

bash
cd "resources/[ui]/lo_ui/web"
npm run build     # tsc && vite build  → web/build/
npm run dev       # Vite localhost:3000 (mode navigateur)
npm run watch     # build incrémental

La sortie va dans web/build/ (écrasée à chaque build). Après un build : restart lo_ui côté serveur.

Structure ​

  • web/src/main.tsx — montage du root React.
  • web/src/App.tsx — monte toutes les sections en permanence ; leur visibilité est pilotée par les events NUI.
  • web/src/ui/ — une section par dossier :
SectionRôle
hudAnneaux santé/endurance/faim/soif, stats monture, états
inventoryInventaire drag & drop (joueur / coffre / magasin / établi), hotbar, menu contextuel
pauseMenu pause (carte, touches, réglages, quitter)
identityFormulaire d'identité / nationalité / apparence
inputBoîtes de dialogue génériques (texte, select, slider, confirm, form)
notifyNotifications toast (info/success/error/warning/money)
lotteryTicket à gratter (canvas)
lockpickingMini-jeu de crochetage
  • web/src/app/AppContext.tsx — contexte global : locale (i18n), imagepath, traducteur t().

Messaging NUI ​

Helpers dans web/src/nui/ :

  • Lua → React : SendNUIMessage({ action, data }) côté Lua → useNuiEvent(action, handler) côté React.
  • React → Lua : fetchNui(name, data, mock) côté React → RegisterNUICallback(name, fn) côté Lua.
tsx
useNuiEvent<HudData>('hud:update', (data) => setHud(data))
const result = await fetchNui('inventory:move', { from, to })

Mode navigateur (dev) ​

isEnvBrowser() est vrai quand window.invokeNative est absent (Vite dev). Dans ce cas, App.tsx monte aussi DevMenu qui injecte des données de test (mocks dans web/src/ui/inventory/mock.ts) pour développer chaque section sans le jeu.

Images d'items ​

Les icônes sont résolues sur nui://lo_ui/web/images/<name>.png. Le chemin de base (imagepath) est transmis depuis Lua via l'event locale et stocké dans AppContext. Le bridge lo_jobscreator utilise le filtre items:imageUrl → nui://lo_ui/web/images/%s.png.
