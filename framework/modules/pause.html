Menu pause ​

Le module pause remplace entièrement le menu pause natif de RedM par un menu custom rendu en NUI. Il intercepte les touches d'ouverture du menu natif, le désactive, et ouvre à la place l'interface lo. Le menu propose les actions de session (fermer, quitter, ouvrir la carte, ouvrir les réglages natifs du jeu) ainsi que la gestion des raccourcis clavier et des réglages joueur exposés par le framework.

L'API est accessible via le global nommé Pause, et un jeu d'exports équivalents est fourni pour les ressources tierces.

Module

Ressource : lo — domaine pause/ (client + server).

Vue d'ensemble ​

Le menu pause natif est neutralisé en continu par un thread client : les contrôles d'ouverture sont désactivés et SetPauseMenuActive(false) est appelé dès que le menu natif tente de s'activer. L'ouverture du menu custom est déclenchée à la place lorsque le joueur appuie sur la touche d'ouverture (avec des délais de grâce pour éviter les ouvertures parasites après une interaction NUI ou la carte).

Le module gère trois états distincts, qui se neutralisent mutuellement :

  • le menu pause custom (Pause:IsOpen)
  • les applications natives uiapp lancées depuis le menu, comme les réglages du jeu (Pause:IsNativeUiappRunning)
  • la vue carte avec prop et animation en main (Pause:IsMapOpen)

Ouverture et fermeture ​

Pause:Open ​

Ouvre le menu pause custom. La fonction est sans effet si le menu est déjà ouvert. Elle prend le focus NUI, joue l'effet de post-traitement PauseMenuIn, joue le son MENU_ENTER puis envoie un message NUI d'ouverture.

lua
Pause:Open({ player = { name = GetPlayerName(Ped:PlayerId()) } })

Le paramètre data est une table optionnelle transmise telle quelle à l'interface NUI via le message { type = 'open', screen = 'pause', data = ... }.

Pause:Close ​

Ferme le menu pause custom, réinitialise le focus NUI, arrête l'effet de post-traitement et joue le son MENU_CLOSE.

lua
Pause:Close()

Pause:Toggle ​

Bascule l'état du menu : ferme s'il est ouvert, ouvre sinon. Accepte la même table data que Pause:Open.

lua
Pause:Toggle({ player = { name = GetPlayerName(Ped:PlayerId()) } })

Pause:IsOpen ​

Renvoie true si le menu pause custom est actuellement ouvert.

lua
if Pause:IsOpen() then
    Pause:Close()
end

Fonctions d'état ​

FonctionRetourDescription
Pause:IsOpen()booleanLe menu pause custom est ouvert.
Pause:IsNativeUiappRunning()booleanUne application native uiapp lancée depuis le menu (réglages du jeu) est en cours.
Pause:IsMapOpen()booleanLa vue carte est active (prop en main ou uiapp map en cours).

Pourquoi ces états

Le thread de suppression du menu natif s'arrête de désactiver les contrôles quand IsNativeUiappRunning ou IsMapOpen renvoient true. Cela laisse le joueur naviguer dans les réglages natifs du jeu ou consulter la carte sans que le système ré-intercepte ses touches.

Exports ​

Le module expose des exports équivalents aux fonctions du Core, plus un export de suppression.

ExportParamètresDescription
OpenPauseMenudata (table, optionnel)Équivaut à Pause:Open(data).
ClosePauseMenu—Équivaut à Pause:Close().
TogglePauseMenudata (table, optionnel)Équivaut à Pause:Toggle(data).
IsPauseMenuOpen—Renvoie true si le menu est ouvert.
SetPauseSuppressedstate (boolean)Active/désactive la suppression du menu pause custom.
lua
exports.lo:OpenPauseMenu({ player = { name = 'John Marston' } })

-- Empêcher l'ouverture du menu pendant une cinématique
exports.lo:SetPauseSuppressed(true)
-- ... puis réactiver
exports.lo:SetPauseSuppressed(false)

SetPauseSuppressed

Quand la suppression est active, le thread n'ouvre plus le menu custom et ne désactive plus les contrôles d'ouverture. Pensez à toujours rétablir l'état à false une fois votre séquence terminée.

Commande ​

CommandeDescription
pausemenuBascule le menu pause custom (équivaut à Pause:Toggle).

La commande transmet le nom du joueur dans la table data :

lua
RegisterCommand('pausemenu', function()
    Pause:Toggle({ player = { name = GetPlayerName(Ped:PlayerId()) } })
end, false)

Touche d'ouverture ​

L'ouverture du menu est déclenchée par le contrôle INPUT_FRONTEND_PAUSE_ALTERNATE (la touche d'échappement / pause). Les trois contrôles suivants sont désactivés en continu pour neutraliser le menu natif :

ContrôleHash
INPUT_FRONTEND_CANCEL0x156F7119
INPUT_FRONTEND_PAUSE_ALTERNATE0x4A903C11
INPUT_FRONTEND_PAUSE0xD82E0BD2

Des délais de grâce empêchent l'ouverture immédiate après certaines actions :

ConstanteValeurRôle
NUI_GRACE_MS200Délai après une perte de focus NUI.
MAP_GRACE_MS300Délai après la fermeture de la carte.

Actions du menu (callbacks NUI) ​

L'interface NUI déclenche les actions du menu via des callbacks. Ces callbacks sont gérés côté client par le module.

Callback NUIEffet
closeMenuFerme le menu pause.
quitGameFerme le menu, puis déclenche la déconnexion de la session après un court délai.
openGameSettingsLance l'application native settings_menu (réglages du jeu).
openMapOuvre la vue carte custom (prop carte en main + animation).
playSoundJoue un son de menu.
dispatchMenuActionAction générique paramétrée (voir ci-dessous).

dispatchMenuAction ​

Callback générique permettant de router une action depuis l'interface. La table reçue contient un champ type et un payload optionnel.

typeComportement
closeMenuFerme le menu pause.
quitGameFerme le menu puis déclenche la déconnexion.
eventDéclenche un event nommé payload.name avec payload.args. Si payload.server est vrai, l'event est serveur (TriggerServerEvent), sinon client (TriggerEvent).

Format du payload pour une action event :

lua
{
    type = 'event',
    payload = {
        name = 'mon:event',
        args = { 'arg1', 42 },
        server = true,
    },
}

Les types inconnus renvoient { ok = false, error = 'unknown_action_type' }.

Quitter la partie ​

L'action quitGame déclenche l'event serveur lo:pausemenu:quit, qui éjecte le joueur de la session.

lua
Net:SecureEvent('lo:pausemenu:quit', function()
    local src = source
    Logger:info('Session', 'Déconnexion via le menu pause : %s', src)
    DropPlayer(tostring(src), 'Vous avez quitté la session.')
end, { maxPerSecond = 2 })

Event sécurisé

lo:pausemenu:quit est enregistré via Net:SecureEvent avec une limite de maxPerSecond = 2. Le message de déconnexion affiché au joueur est « Vous avez quitté la session. ».

Vue carte ​

L'action openMap ouvre une vue carte immersive : le menu pause se ferme, un prop carte est attaché à la main du joueur avec une animation de consultation, puis l'application native map est lancée. La vue reste active tant que l'uiapp map tourne ; à sa fermeture, le prop et l'animation sont retirés automatiquement.

ÉlémentValeur
Dictionnaire d'animationmech_inspection@mini_map@base
Animationhold
Modèle du propmp001_mp_map01x
Bone d'attacheXH_L_Hand00

Gestion des raccourcis clavier ​

Le menu pause expose l'édition des raccourcis clavier du framework (domaine input, global Keys) via des callbacks NUI. Seuls les bindings autorisés (Keys:IsAllowed) et non verrouillés sont exposés. La catégorie Armurerie est verrouillée et ne peut être ni modifiée ni réinitialisée depuis le menu.

Callback NUIEffet
keys:listRenvoie la liste des bindings autorisés.
keys:setAffecte une touche à un binding (resource, name, key).
keys:resetRéinitialise un binding à sa valeur par défaut (resource, name).
keys:resetAllRéinitialise tous les bindings autorisés.
keys:captureDémarre la capture d'une touche pressée par le joueur.
keys:cancelCaptureAnnule une capture en cours.

Lors de la capture (keys:capture), les touches Échap (0x1B) et Retour arrière (0x08) annulent l'opération. Les codes virtuels des boutons souris sont ignorés. La capture renvoie le code virtuel vk et son nom d'affichage via Keys:GetDisplayName.

Chaque binding transmis à l'interface est normalisé avec les champs suivants :

Champ NUISourceDescription
idbinding.keyIdentifiant du binding.
resourcebinding.resourceRessource propriétaire.
namebinding.nameNom du binding.
vkbinding.vkCode virtuel actuel.
keyNamebinding.keyNameNom de la touche actuelle.
defaultVKbinding.defaultCode virtuel par défaut.
defaultNamebinding.defaultNameNom de la touche par défaut.
descriptionbinding.descriptionDescription du binding.
categorybinding.categoryCatégorie d'affichage.
levelbinding.levelNiveau requis.

Gestion des réglages joueur ​

Le menu pause expose également les réglages joueur (global Settings) via des callbacks NUI. Les réglages sont persistés par joueur via les KVP de la ressource.

Callback NUIEffet
settings:listRenvoie la liste des réglages (Settings.List).
settings:setModifie un réglage (id, value).
settings:resetRéinitialise un réglage à sa valeur par défaut (id).
settings:resetAllRéinitialise tous les réglages.

Config.Pausemenu ​

La configuration du menu pause se trouve dans config.lua :

lua
Config.Pausemenu = {
    Settings = {},
}
ChampTypeDescription
SettingstableListe des définitions de réglages enregistrées au démarrage dans le menu pause.

Chaque entrée de Config.Pausemenu.Settings est une définition de réglage chargée par le global Settings. Une définition valide doit comporter un id (string), un type parmi toggle, slider ou select, et une valeur par défaut default.

Clé de définitionTypeObligatoireDescription
idstringOuiIdentifiant unique du réglage.
typestringOuitoggle, slider ou select.
defaultanyOuiValeur par défaut.
categorystringNonCatégorie d'affichage (general par défaut).
minnumberPour sliderBorne minimale.
maxnumberPour sliderBorne maximale.
stepnumberNonPas du slider.
unitstringNonUnité affichée.
optionstablePour selectListe des valeurs possibles.
applyfunctionNonFonction appelée à chaque changement de valeur.

Exemple de définition de réglage :

lua
Config.Pausemenu = {
    Settings = {
        {
            id = 'hud_minimap',
            type = 'toggle',
            category = 'general',
            default = true,
            apply = function(value)
                -- appliquer l'état au HUD
            end,
        },
        {
            id = 'voice_range',
            type = 'slider',
            category = 'audio',
            default = 2,
            min = 1,
            max = 3,
            step = 1,
        },
    },
}

Réglages dynamiques

D'autres ressources peuvent enregistrer leurs propres réglages au runtime via les exports RegisterSetting, GetSetting, SetSetting et RemoveSetting du framework. Ils apparaîtront automatiquement dans le menu pause.

Comportement à l'arrêt de la ressource ​

Si la ressource lo est arrêtée alors que le menu est ouvert, le focus NUI est réinitialisé pour éviter de bloquer l'écran du joueur.
