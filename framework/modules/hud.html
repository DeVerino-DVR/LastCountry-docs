Module HUD ​

Le module hud gère l'affichage tête haute du joueur : santé, endurance, cores, faim, soif, indicateur vocal de proximité, infos de monture, états visuels (blessé, épuisé, suralimenté), ainsi que la minimap, le mode AFK et le métabolisme (perte progressive de faim/soif). Il est rendu via une interface NUI alimentée par des messages SendNUIMessage, et expose son API à la fois en exports et sur le global nommé Hud. Cette page documente uniquement ce qui existe réellement dans le domaine et dans Config.Hud.

Ce que le HUD affiche ​

À chaque tick (toutes les Config.Hud.UpdateMs millisecondes), le client collecte l'état du joueur et de sa monture, puis envoie un message NUI update contenant un objet data. Le HUD n'est ré-émis que si la signature des valeurs a changé depuis le dernier tick, afin d'éviter le spam NUI.

Champs envoyés à l'interface dans le message update :

ChampTypeSource
hiddenboolVrai si AFK, masqué manuellement, menu ouvert, pause, carte ouverte, focus NUI ou caméra cinématique
healthnumber (0-100)Santé du ped (GetEntityHealth / GetEntityMaxHealth)
staminanumber (0-100)Endurance du ped
healthCorenumber (0-100)Core de santé (index 0)
staminaCorenumber (0-100)Core d'endurance (index 1)
hungernumber (0-100)Faim (state bag hunger)
thirstnumber (0-100)Soif (state bag thirst)
voicenumber (0-100)Indicateur vocal de proximité (voir plus bas)
voiceTalkingboolVrai si le joueur parle (MumbleIsPlayerTalking)
staminaActiveboolVrai si l'endurance décroît (delta > 0.3)
horseStaminaActiveboolIdem pour la monture

horseStaminanumber (0-100)Endurance de la monture

Cores RDR2

Le module force en continu (toutes les secondes) l'affichage des cores natifs à 2 pour les 12 attributs, afin que les valeurs lues soient cohérentes avec celles rendues par le HUD personnalisé.

Indicateur vocal de proximité ​

La barre voice est calculée à partir de la distance de proximité vocale du joueur, normalisée sur Config.Hud.Voice.maxRange :

lua
local proximity = LocalPlayer.state and LocalPlayer.state.proximity
if proximity and proximity.distance then
    voicePct = clamp(proximity.distance / maxRange * 100.0)
end

Sans donnée de proximité, la valeur par défaut est 100.0. Le drapeau voiceTalking indique si le joueur est en train de parler.

Métabolisme (faim / soif) ​

La faim et la soif sont stockées dans les state bags joueur hunger et thirst (valeurs 0-100). Un planificateur dédié décrémente ces valeurs à intervalle régulier.

lua
Scheduler:Register('hud:metabolism', M.interval or 60000, function()
    if PlayerData:IsLoaded() and not IsCinematicCamRendering() then
        setNeed('hunger', getNeed('hunger') - (M.hungerRate or 1.0))
        setNeed('thirst', getNeed('thirst') - (M.thirstRate or 1.5))
        TriggerServerEvent('lo:hud:updateMeta', collectMeta())
    end
end)

À chaque cycle, le client envoie lo:hud:updateMeta au serveur avec un instantané (hunger, thirst, et si le ped est vivant health, stamina). Le serveur stocke ces valeurs dans les state bags et dans metadata du joueur.

Persistance serveur ​

Le domaine hud est un domaine plat (hud/server/hud.lua, hud/client/*.lua) chargé via le fxmanifest.lua : il n'y a pas de registre de modules. La persistance faim/soif s'accroche aux hooks de cycle de vie exposés par le noyau (shared/lifecycle.lua) :

  • Lifecycle.OnPlayerLoaded(fn) : initialise les state bags hunger et thirst depuis metadata (défaut 100).
  • Lifecycle.OnPlayerSave(fn) : recopie les state bags hunger/thirst dans metadata.
lua
Lifecycle.OnPlayerLoaded(function(player)
    local src = player.source
    local metadata = player.PlayerData.metadata or {}
    Player(src).state.hunger = tonumber(metadata.hunger) or 100
    Player(src).state.thirst = tonumber(metadata.thirst) or 100
end)

Lifecycle.OnPlayerSave(function(player)
    local state = Player(player.source).state
    if type(state.hunger) == 'number' then player.PlayerData.metadata.hunger = state.hunger end
    if type(state.thirst) == 'number' then player.PlayerData.metadata.thirst = state.thirst end
end)

L'event entrant est sécurisé contre le spam :

lua
Net:SecureEvent('lo:hud:updateMeta', function(data)
    -- ...
end, { maxPerSecond = 3 })

Les valeurs reçues côté serveur sont bornées par clampVital (0-100, arrondies).

Compatibilité métabolisme externe ​

Le module écoute aussi l'event vorpmetabolism:changeValue (clés Hunger / Thirst) et convertit la valeur reçue avant de mettre à jour les state bags, ce qui permet d'alimenter le HUD depuis une source de métabolisme tierce.

États ​

Quand Config.Hud.States.enabled est vrai, le tick calcule une table states envoyée au HUD. Chaque état peut être automatique (déduit des seuils de config) ou forcé manuellement (via setState). Un état forcé écrase toujours la condition automatique.

ÉtatCondition automatiqueSeuil de config
woundedhealth inférieur au seuilStates.woundedBelow
drainedstamina inférieur au seuilStates.drainedBelow
overfedhunger supérieur au seuilStates.overfedAbove
colduniquement forcé—
hotuniquement forcé—
horse_dirtyuniquement forcé—
agitationuniquement forcé—
lua
local function stateActive(name, autoCond)
    local forced = forcedStates[name]
    if forced ~= nil then return forced end
    return autoCond == true
end

Les états cold, hot, horse_dirty et agitation n'ont pas de condition automatique dans le module : ils ne s'activent que si vous les forcez via setState (ou Hud.SetState).

Minimap ​

La minimap possède trois modes, sélectionnables via un groupe de prompts affiché en bas à droite lorsque le joueur maintient INPUT_HUD_SPECIAL. Le mode choisi est persisté en KVP (lo:hud:mapmode).

ToucheLibelléModeEffet
UPNormale1Radar rond affiché, layout round
RIGHTBoussole3Radar plat affiché, layout flat
DOWNCacher0Radar masqué, layout flat

La minimap peut aussi être activée/désactivée globalement (message NUI minimap) via setMinimap ou l'event lo:hud:setMinimap. L'état initial vient de Config.Hud.Minimap.enabled.

Repositionnement du HUD ​

Le module permet de déplacer le HUD à la souris. Les ancres sont stockées séparément pour le layout round et flat, et persistées en KVP (lo:hud:anchor).

CommandeEffet
/hudmouvActive/désactive le mode positionnement (focus NUI souris)
/hudresetRéinitialise les ancres aux positions par défaut et supprime le KVP

Les positions par défaut sont round = { x = 170, y = 155 } et flat = { x = 50, y = 20 }. La sauvegarde se fait via le callback NUI savePos.

AFK ​

Quand Config.Hud.Afk.enabled est vrai, le tick surveille les déplacements et les mouvements de caméra du joueur. S'il n'y a aucune activité pendant Config.Hud.Afk.timeout millisecondes (et que le mode positionnement n'est pas actif), le HUD est masqué (hidden = true).

lua
if #(coords - lastCoords) > 0.15 or math.abs(cam - lastCam) > 0.5 then
    lastActive = GetGameTimer()
end
afk = (GetGameTimer() - lastActive) > (A.timeout or 10000) and not positioning

L'activité est détectée par un déplacement de plus de 0.15 unité ou une rotation de caméra de plus de 0.5.

Configuration ​

Tous les réglages se trouvent dans Config.Hud.

lua
Config.Hud = {
    UpdateMs = 250,
    Minimap = {
        enabled = true,
    },
    Voice = {
        maxRange = 18.0,
    },
    States = {
        enabled = true,
        woundedBelow = 30,
        drainedBelow = 15,
        overfedAbove = 96,
    },
    Metabolism = {
        interval = 60000,
        hungerRate = 1.0,
        thirstRate = 1.5,
    },
    Afk = {
        enabled = true,
        timeout = 10000,
    },
}

Voice.maxRange18.0Portée maximale (mètres) utilisée pour normaliser l'indicateur vocal

States.woundedBelow30Seuil de santé sous lequel l'état wounded s'active automatiquement
States.drainedBelow15Seuil d'endurance sous lequel l'état drained s'active automatiquement

API ​

L'API est disponible à la fois via exports et sur l'objet Hud (client).

FonctionHudDescription
setStatus(name, pct)SetStatusDéfinit hunger/faim ou thirst/soif à un pourcentage
getHunger()GetHungerRetourne la faim actuelle (0-100)
getThirst()GetThirstRetourne la soif actuelle (0-100)
addHunger(v)AddHungerAjoute v à la faim (peut être négatif)
addThirst(v)AddThirstAjoute v à la soif (peut être négatif)
setMinimap(state)SetMinimapActive/désactive la minimap
setState(name, active)SetStateForce (ou libère) un état visuel
setVisible(visible)SetVisibleAffiche/masque le HUD manuellement

Exemple d'appel via exports :

lua
exports.lo:addThirst(-10)         -- retire 10 de soif
exports.lo:setState('cold', true) -- force l'état "cold"
exports.lo:setVisible(false)      -- masque le HUD

Exemple d'appel via Hud :

lua
Hud.SetStatus('faim', 100)
local h = Hud.GetHunger()
