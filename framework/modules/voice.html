Module Voix ​

Le module voice gère la voix de proximité du framework lo par-dessus le système Mumble natif de FiveM. Il assigne automatiquement un canal Mumble à chaque joueur, gère les modes de portée (chuchoter / normal / crier), le cycle de portée via une touche dédiée, l'écoute en mode spectateur, les appels téléphoniques (regroupement de joueurs sur un canal commun) ainsi que le mute administratif temporisé.

Le module est compatible avec l'API d'exports de pma-voice : les exports et events setPlayerCall, addPlayerToCall, removePlayerFromCall sont relayés, ce qui permet aux ressources tierces de fonctionner sans modification.

Localisation

Code source : resources/[core]/lo/voice/client/voice.lua et resources/[core]/lo/voice/server/voice.lua. Configuration : Config.Voice dans resources/[core]/lo/config.lua.

Modes de proximité ​

Chaque mode est défini par un couple [ portée, libellé ]. La portée est exprimée en mètres et appliquée via MumbleSetTalkerProximity. Trois modes sont fournis par défaut.

IndexPortée (m)LibelléDescription
13.0ChuchoterVoix basse, courte portée
28.0NormalPortée standard
318.0CrierVoix forte, longue portée

Au démarrage, le mode sélectionné est celui défini par Config.Voice.DefaultMode (par défaut l'index 2, soit Normal). À chaque changement de mode, une notification est affichée au joueur via Notify :

lua
Notify(('Voix : %s'):format(modes[mode][2]))

L'état de proximité courant est exposé sur le statebag du joueur local sous la clé proximity, ce qui permet à un HUD de l'afficher :

lua
-- Structure du statebag local 'proximity'
LocalPlayer.state.proximity = {
    index = 2,          -- index du mode courant
    distance = 8.0,     -- portée en mètres
    mode = 'Normal',    -- libellé du mode, ou 'Custom' si surchargé
}

Audio natif

Lorsque le convar voice_useNativeAudio vaut true, la portée effective d'écoute des joueurs proches est multipliée par 3 en interne pour rester cohérente avec le rendu audio natif. Côté serveur, si aucun convar audio (voice_useNativeAudio, voice_use2dAudio, voice_use3dAudio) n'est défini, le module active automatiquement voice_useNativeAudio et voice_useSendingRangeOnly.

Cycle de portée (touche F1) ​

Le module enregistre une touche permettant de passer d'un mode au suivant. La touche est définie par Config.Voice.CycleKey (par défaut F1) et enregistrée via Keys:RegisterKey dans la catégorie Voix :

lua
local cycleKey = Keys:RegisterKey('voice_proximity', 'Changer la portée de la voix', Config.Voice.CycleKey or 'F1', 'Voix')
if cycleKey then
    cycleKey:onPressed(cycleProximity)
end

À chaque appui, le mode passe à l'index suivant, et revient à 1 après le dernier mode (rotation circulaire). Le cycle est ignoré tant qu'une surcharge de portée est active (voir plus bas). À chaque changement, l'event client lo:voice:modeChanged est déclenché.

Event lo:voice:modeChanged ​

Déclenché côté client à chaque changement de mode via le cycle.

ArgumentTypeDescription
indexnumberIndex du nouveau mode
labelstringLibellé du nouveau mode
lua
AddEventHandler('lo:voice:modeChanged', function(index, label)
    print(('Nouveau mode vocal : %s (#%d)'):format(label, index))
end)

Exports client ​

overrideProximityRange ​

Force une portée vocale personnalisée (en mètres), indépendante des modes. Tant qu'une surcharge est active, le cycle via la touche est désactivé et le libellé exposé devient Custom.

lua
exports.lo:overrideProximityRange(25.0)
ParamètreTypeDescription
rangenumberPortée en mètres à appliquer

clearProximityOverride ​

Annule la surcharge de portée et restaure le mode courant.

lua
exports.lo:clearProximityOverride()

getVoiceMode ​

Retourne le mode vocal courant sous la forme d'une table.

lua
local data = exports.lo:getVoiceMode()
-- data.index  -> index du mode courant
-- data.label  -> libellé du mode courant

setCallChannel / addPlayerToCall / removePlayerFromCall ​

Gèrent l'appartenance du joueur local à un canal d'appel. setCallChannel et addPlayerToCall prennent un numéro de canal ; removePlayerFromCall retire le joueur de son appel (équivalent au canal 0).

lua
exports.lo:setCallChannel(101)   -- rejoint le canal d'appel 101
exports.lo:removePlayerFromCall() -- quitte l'appel courant

Appels (côté serveur) ​

Le module regroupe les joueurs partageant un même canal d'appel afin qu'ils s'entendent quelle que soit leur distance. Le volume des voix d'appel est fixé par Config.Voice.CallVolume via MumbleSetVolumeOverrideByServerId.

Export setPlayerCall ​

Assigne un joueur à un canal d'appel. Passer le canal 0 retire le joueur de son appel courant.

lua
exports.lo:setPlayerCall(src, 101) -- ajoute le joueur src à l'appel 101
exports.lo:setPlayerCall(src, 0)   -- retire le joueur src de l'appel
ParamètreTypeDescription
srcnumberServer ID du joueur
channelnumberCanal d'appel (>= 0, 0 pour quitter)

Event sécurisé lo:voice:setPlayerCall ​

Le client déclenche cet event serveur (via Net:SecureEvent, limité à 5 appels par seconde) pour modifier son propre canal d'appel.

lua
TriggerServerEvent('lo:voice:setPlayerCall', 101)

Events de synchronisation client ​

EventDirectionDescription
lo:voice:syncCallDataS -> CEnvoie la liste complète des participants de l'appel
lo:voice:addPlayerToCallS -> CUn joueur rejoint l'appel
lo:voice:removePlayerFromCallS -> CUn joueur quitte l'appel

Mute administratif (côté serveur) ​

Un joueur peut être coupé temporairement. Le mute est automatiquement levé à expiration de sa durée (vérification chaque seconde). La durée par défaut provient de Config.Voice.MuteDuration.

Export setVoiceMuted ​

lua
exports.lo:setVoiceMuted(src, true, 600) -- mute src pendant 600 secondes
exports.lo:setVoiceMuted(src, false)     -- démute src immédiatement
ParamètreTypeDescription
srcnumberServer ID du joueur
mutedbooleantrue pour couper, false pour rétablir
durationnumberDurée en secondes (optionnel ; défaut MuteDuration)

Retourne true si l'opération a réussi, false si le joueur n'existe pas.

Export isVoiceMuted ​

lua
local muted = exports.lo:isVoiceMuted(src) -- true si le joueur est actuellement coupé

Event lo:voice:playerMuted ​

Déclenché côté serveur à chaque changement d'état de mute.

ArgumentTypeDescription
srcnumberServer ID du joueur
mutedbooleanNouvel état de mute

Commande /muteply ​

Coupe ou rétablit la voix d'un joueur. Réservée à la console (source == 0) ou aux joueurs disposant du niveau de permission 5 (Permissions:HasLevel).

bash
/muteply [id] [durée en secondes]

L'appel bascule l'état : si le joueur ciblé n'est pas coupé, il est mute pour la durée indiquée (ou MuteDuration par défaut) ; s'il est déjà coupé, il est rétabli.

Mode spectateur ​

Lorsqu'un joueur passe en mode spectateur (NetworkIsInSpectatorMode) ou utilise une caméra de rendu (GetRenderingCam() ~= -1), le module ajoute automatiquement l'écoute des canaux des autres joueurs afin d'entendre toutes les conversations, puis la retire au retour en jeu. Ce comportement est entièrement automatique et ne nécessite aucune configuration.

Configuration ​

Bloc Config.Voice (dans resources/[core]/lo/config.lua) :

lua
Config.Voice = {
    Modes = {
        { 3.0, 'Chuchoter' },
        { 8.0, 'Normal' },
        { 18.0, 'Crier' },
    },
    DefaultMode = 2,
    RefreshRate = 200,
    CycleKey = 'F1',
    MuteDuration = 900,
    CallVolume = 0.6,
}
CléTypeDéfautDescription
Modestable3 modesListe des modes [ portée_m, libellé ] parcourus par le cycle
DefaultModenumber2Index du mode sélectionné au spawn (min 1)
RefreshRatenumber200Intervalle (ms) de mise à jour de la portée et des joueurs proches
CycleKeystring'F1'Touche de changement de portée
MuteDurationnumber900Durée par défaut du mute administratif, en secondes
CallVolumenumber0.6Volume des voix d'appel (0.0 à 1.0)

Cohérence des modes

DefaultMode doit pointer vers un index valide de la table Modes. Avec la configuration par défaut, les valeurs admises sont 1, 2 ou 3. Une valeur hors plage ne correspondrait à aucun mode défini.
